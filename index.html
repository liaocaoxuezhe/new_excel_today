历史回顾

1978年，Dan Bricklin和Bob Frankston发明了人类历史上第一个电子表格VisCalc。1985年，微软推出了Excel电子表格软件，从此微软靠着Word、Excel、PowerPoint三件套打遍天下无敌手，至今仍然是市场占有率第一的效率工具。

进入21世纪20年代，以飞书多维表格为代表的新一代文档工具，正在慢慢成为市场的主流。

进入信息时代以后，信息变得越来越杂乱，越来越多，人均处理的"信息"每年都在呈指数级地上涨。

表格的诞生，是因为"表格"是一个最适合人类用来将杂乱的信息结构化，利于后续处理的工具。所以在文本信息和数字为主的时代，Excel是当之无愧的霸主。

Traditional Excel Interface

但在今天，如果要在Excel里放一张图片、放一段音频，或者添加一个PDF附件，都非常膈应。Excel更适合用于存储和展现文本和数字。在信息越来越复杂、越来越多模态的时代，就显得有些捉襟见肘了。

多维表格：信息时代的新答案

当下的信息形态，需要更多维度的存储和展现的方式。

因此，就有了「飞书多维表格」。多维表格，顾名思义，让你可以存放更多维度的内容。前面见到的痛点，在飞书多维表格里，将变得如此自然。

Feishu Base Interface

多维表格本质上是一个数据库

如果深究多维表格的技术原理的话，其实多维表格更像是一个可视化的数据库。作为一个数据库，它天生就支持多种类型的数据。文本、选项、图片、附件、日期…

Multiple Data Types Support

灵活的视图系统

在飞书多维表格里，你可以非常灵活地组织数据的展现形式，看板、日历、甘特图、画册等等视图，你可以根据任何条件和排序来筛选内容，随意切换查看和展现数据的方式。

  • 表格视图：像Excel一样的使用方式，平铺地将内容展开。
  • 看板视图：你可以按照任意一列中的数据，将数据进行分类。
  • 日历视图：按照一个日期的字段，将信息在日历中展示。
  • 甘特视图：适合用来做项目进度管理，项目经理的老朋友了。
  • 画册视图：很适合用来展示图片为主的信息，如商品图等。
  • 表单视图：用于收集表单，用户提交数据后，数据会自动记录在多维表格中。
Multiple View Options

个人项目中的多维表格玩法

飞书多维表格很适合用来做个人的创作管理。我将我的选题思路都维护在多维表格中，把创作分为「选题思路-创作中-待发布-已发布-已归档」几个状态。

每当有新的灵感或选题，就创建一条数据，按照状态进行管理。在飞书多维表格中，你可以轻松切换到看板视图，清晰查看每个选题的进展。
Content Management Kanban

之前我更新了50期的Newsletter，因为和其他选题有比较明显的区分，为了不互相干扰。我就用一个表格视图单独管理起来了，再把这些数据从其他视图中筛选掉，集中查看，清晰又高效。
